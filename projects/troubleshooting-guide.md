# Troubleshooting Framework

##  Standard Method

1. What is happening?
2. Why is it happening?
3. Can I prove it?
4. Can I reproduce it?
5. Can I fix it cleanly?
6. Can I prevent it?

---

##  Usage

Always:
- Describe what you SEE (not assumptions)
- Validate using CLI/logs
- Confirm repeatability
- Apply minimal fix
- Verify result

---

##  Reminder

Explain:
✔ system behavior  
✔ CLI output  
✔ observable results  

Not:
 abstract wording  
 assumptions  
 theory without proof
